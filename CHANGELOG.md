# Changelog

本文件按发布时间倒序记录 Another Atom 已发布的产品版本。只记录已经进入 `main` 的功能；未提交修改、设计 TODO 和 V2 规划不计入已发布能力。

## 2026-07-14 — v0.1.0

- **产品里程碑：** V1.0 可部署纵切版
- **功能基线：** `7156045`
- **版本来源：** `pyproject.toml`、`studio/package.json`
- **Git tag：** 尚未创建

### 本次发布

- **完成从想法到发布的产品闭环**
  - 用户可以通过自然语言提出目标，经确认、构建、预览、修改、版本管理后，手动发布为公开结果。
- **完成固定多角色 Agent 流水线**
  - Product Manager、Architect、Engineer、Data Analyst、Reviewer 按固定顺序交付；Runtime Validator 提供确定性校验。
  - Blueprint、ArchitectureSpec、源码、DataProfile、ValidationReport 和 ReviewReport 可以被用户检查。
- **完成自包含 Web 应用生成**
  - 支持生成 HTML、CSS、JavaScript 网页应用，并在桌面和移动尺寸下进行真实交互预览。
  - V1 保留用户提出的产品类型，不默认把游戏或工具改写成商品目录。
- **完成 Project、源码与版本管理**
  - Project 保存源码和生成产物；Build、Edit、Restore 形成可追溯版本，恢复旧版本不会删除历史。
  - 发布版本与工作版本分离，新构建不会自动覆盖线上结果。
- **完成账号级项目隔离**
  - Project、Run、Preview、Repository 和版本按登录用户校验归属。
- **完成持久化执行与恢复**
  - 构建在后台执行，服务恢复后复用已完成阶段；校验失败时可以自动修复并重新校验。
  - 阶段进度、运行事件和调试日志可查看，运行日志可下载并持久化。
- **完成真实模型接入与受控降级**
  - 支持真实 LLM Provider；超时或结构化响应失败时执行有限重试、修复或已配置的 Provider 降级，并记录过程。
- **完成只读管理员后台**
  - 管理员可以搜索用户，查看 Project、运行状态和日志；普通用户及未登录请求不能访问管理 API。

### 主要修复

- 加固单实例任务恢复，避免服务重启后重复生成、重复结算或重复创建版本。
- 增加独立 Reviewer，并在确定性校验失败后进入自动修复流程。
- 修复登录校验错误反馈不及时、模型输出异常和 Preview 状态不清晰的问题。
- 持久化进程日志，并提供运行日志下载。

### 验证结果

- 2026-07-13：85 项后端自动化测试通过。
- Studio 代码检查与生产构建通过。
- Railway 单副本、持久化磁盘和公开访问已完成验收。
- 管理员后台已通过仓库内自动化验证；非默认管理员凭据与真实持久化数据的 Railway 环境验收尚未完成。

### 当前边界

- 通用 Human-in-the-loop Approval 和 PM 产品文档确认流程尚未实现；当前只有 Blueprint 适配版审批。
- 基于现有项目文件持续对话、审阅 Diff、落盘修改并创建版本的完整 AI Coding 闭环尚未实现。
- 在线 Vim/Sandbox 已有仓库和接口基础，但真实 Linux 隔离尚未完成目标 Host 安全验收。
- 不支持生成依赖服务端认证、支付、云数据库、第三方 API、原生 Runtime 或任意 Shell 的应用。
- V2 动态 TaskGraph、Agent 并行、返工和仲裁尚未实现。
