# V1 简要交付说明

## 产品做什么

Another Atom 把用户的一句话需求变成一个可以预览、修改、保存版本并发布的 Web 应用。

用户可以提出游戏、工具、看板或商品目录等不同目标，系统默认保留这个目标，而不是把它改写成另一类产品。产品经理会先把需求整理成一份可确认的方案；确认后，固定团队依次完成架构、代码和校验。每一步都有可查看的结果，而不是只展示一段 AI 对话。

V1 的交付方式是自包含的 HTML、CSS 和 JavaScript 网页应用。它不是任意代码执行平台：真实登录、支付、云数据库、第三方服务、原生应用和通用终端暂不在范围内；遇到这类需求，系统会说明需要调整或暂时无法构建。

## 现在可以体验什么

- **[从需求到预览]** 登录后可与 Lead 对话，由产品经理生成需求草案；确认后进入固定团队构建，并查看构建进度、运行日志和结果预览。
- **[查看和修改]** 可查看本次生成的源码文件与过程产物，修改内容或样式，保存为新的版本。
- **[版本与发布]** 每次 Build、Edit 或 Restore 都会形成新版本；是否上线、上线哪个版本，始终由用户手动决定。
- **[可靠性]** 项目、构建过程、版本和用量会保存下来；服务重启后会继续已保存的工作，避免重复生成和重复扣费。
- **[在线部署]** Railway 单副本、持久化磁盘和公开链接已经完成实际验收。

目前后端自动化测试 73 项通过，Studio 的代码检查和生产构建通过。

## 目前还没有做完

- **[真实隔离环境]** Vim 和 Sandbox 的产品入口已经有了，但 Linux 隔离、禁网、资源限制和清理机制还需要在目标机器上验证。
- **[完整项目对话]** Lead 的澄清、团队构建和后续修改尚未串成一条可完整回看的项目对话记录。
- **[外部服务]** 不支持真实 OAuth、支付、云数据库、第三方 API、任意后端代码或通用 Shell。
- **[大规模部署]** 当前只支持单实例运行，不支持多 Worker、多副本或分布式调度。

## 后续优先级

1. **[P0｜完成真实隔离环境]** 在目标 Linux Host 验证 Vim/Sandbox 的隔离、禁网、资源限制和清理机制。
2. **[P0｜让用户能顺畅继续]** 补全项目对话、失败后的重试和修复流程，让用户知道系统为什么停下、可以怎样继续。
3. **[P1｜增强网页应用能力]** 在不开放任意执行权限的前提下，逐步补充组件、交互和多页面状态。
4. **[P2｜升级多 Agent 协作]** 在 V1 的部署和安全边界稳定后，再做动态任务拆分、并行、返工和仲裁。

## 总结

V1 已经跑通了从需求、构建、预览、修改、版本到手动发布的闭环，并完成 Railway 部署验收。下一步最重要的不是增加更多 Agent，而是完成真实 Sandbox 验收和用户可继续的对话、重试与修复体验。
