# Another Atom 讨论文档规范与索引

[toc]

## 1. 作用

`docs/discussion/` 保存尚未形成正式决策的结构化讨论，回答“问题是什么、有哪些方案、各自代价是什么、当前倾向和未决项是什么”。它不是聊天记录、实现基线或验收证据。

Discussion 不按版本建目录，因为“属于 V1、V2 还是整体”可能正是讨论内容。版本范围写在文档元信息中：`V1`、`V2`、`整体` 或 `待定`。

## 2. 领域

- [产品讨论](./产品讨论/README.md)：产品目标、用户流程、交互、范围和价值取舍；
- [Agent 讨论](./Agent讨论/README.md)：Context、Memory、角色、调度、Tool 和 Sandbox 权限取舍；
- [工程讨论](./工程讨论/README.md)：架构、数据、执行环境、部署和可靠性选项；
- [综合讨论](./综合讨论/README.md)：确实跨越前三类且无法由一个主要问题归属的讨论。

## 3. 文件规则

- 文件名使用 `YYYY-MM-DD-中文短主题.md`；
- 首屏写清状态、日期、版本范围和关联设计；
- 正文保留真实问题、已知事实、备选方案、优势、代价、适用边界、当前倾向和未决问题；
- 不复制完整聊天，不把没有依据的设想写成事实；
- 只有会影响范围、Contract、权限或高成本取舍的讨论才需要存档。

## 4. 收敛规则

讨论形成决定后：

1. 在对应 `docs/design/{V1|V2|整体}/` 创建或更新正式设计；
2. Discussion 状态改为 `已收敛`；
3. 链接正式设计并保留未采用方案及原因；
4. 后续 Contract 只在 Design 中维护。

```text
Discussion 未决讨论
  -> Design 正式决定
  -> 实现
  -> Review 证据检查
  -> 新问题回到 Discussion 或 Design
```
