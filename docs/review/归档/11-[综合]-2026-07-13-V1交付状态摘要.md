## 产品要解决什么

Another Atom 不是把一句 Prompt 换成一张一次性的网页截图。它要解决的是：用户有一个模糊想法时，如何让 AI 把它变成一个自己看得懂、能继续修改、能保留历史并由自己决定是否上线的网页应用。

产品的核心链路是：**说清想法 → 确认方案 → 生成应用 → 查看和修改 → 保存版本 → 手动发布**。

用户可以提出游戏、工具、看板或商品目录等不同目标，系统默认保留这个目标，而不是把它改写成另一类产品。V1 交付的是自包含的 Web 应用；真实支付、云数据库、第三方服务、原生应用和通用终端不假装已经支持。

## 已实现

- **[主线：从想法到可发布项目]** 用户向 Lead 说出要做什么，产品经理先整理需求方案；随后 Architect、Engineer、Data Analyst 和 Reviewer 分别完成结构、实现、数据分析和独立审查，Runtime Validator 提供确定性证据。用户拿到可交互预览后，可以修改、保存版本、恢复历史并手动发布。

- **[先对齐，再构建]** Lead 负责区分提问和明确构建请求。进入构建后，产品经理先给出可编辑的需求方案；可直接在浏览器中完成的工作自动继续，只有范围变化、额外预算或上线等真实风险才需要用户确认。

- **[让 AI 的工作可检查]** Product Manager、Architect、Engineer、Data Analyst、Reviewer 按固定顺序交接。用户能查看需求方案、架构、源码、DataProfile、确定性校验和 ReviewReport，而不是只能等待一个黑箱结果。

- **[把生成结果变成自己的项目]** 每次构建、编辑或恢复都会留下源码和独立版本。用户可以预览、修改、回到旧版本；生成完成不会自动上线，发布哪个版本由用户手动决定。

- **[让一次构建可以继续]** 构建在后台执行，页面会展示当前进度和日志。项目、阶段结果、模型用量和版本都会保存；服务重启后会接着已有结果继续，避免重复生成和重复扣费。

- **[真实模型、真实在线链路]** 产品实际调用 DeepSeek V4；Ollama Cloud 超时且配置了官方 Key 时，会切换到 DeepSeek 官方 API，并把切换过程写入日志。Railway 单副本、持久化磁盘和公开访问已经完成验收。

- **[项目归用户所有]** 登录状态、项目、预览、源码和版本都按用户隔离；一个用户不能通过前端参数访问另一个用户的项目。

截至 2026-07-13，项目虚拟环境执行 `.venv/bin/pytest -q`，85 项后端自动化测试通过；Studio 的代码检查和生产构建通过。

## 未实现

- **[完整的项目对话]** Lead 的澄清、团队构建和后续修改还没有串成一条可完整回看的项目对话记录。用户目前能看到构建产物和日志，但不能像聊天软件一样回放整个项目讨论。

- **[真实隔离编辑环境]** Vim 和 Sandbox 的入口已经有了，但 Linux 隔离、禁网、资源限制和清理机制还需要在目标机器完成验证后，才能作为可放心使用的在线编辑能力。

- **[需要后端的产品能力]** 真实 OAuth、支付、云数据库、第三方 API、任意后端代码和通用 Shell 仍不支持。V1 先保证纯 Web 应用的交付闭环，不把未落地的外部能力包装成已完成。

- **[大规模协作 Runtime]** 当前只支持单实例、固定团队顺序。多 Worker、多副本，以及动态拆任务、并行、返工和仲裁属于 V2。

## 后续优先级

1. **[P0｜补齐 AI Coding 修改闭环]** 用户可以围绕当前项目持续对话，查看和选择文件，提出修改要求或审阅 Agent 的修改建议；确认后的变更要真实写入源码，展示差异，重新校验并创建版本，而不是只生成一段无法落地的回复。
2. **[P0｜让失败能从原处继续]** 补全失败后的重试、修复和恢复入口；用户要能理解系统为什么停下，保留已经完成的 Artifact，并从失败阶段继续，而不是重新创建项目或整轮生成。
3. **[P0｜完成安全编辑闭环]** 在目标 Linux Host 验证 Sandbox 的真实隔离，让源码编辑从“有入口”变成可信能力。
4. **[P1｜增强网页应用本身]** 在不开放任意执行权限的前提下，补充组件、交互、多页面状态和更丰富的产品表达。
5. **[P2｜升级协作方式]** 在 V1 稳定后，再做动态拆任务、并行、返工和仲裁，让团队从固定接力升级为更自主的协作。

## 总结

V1 已经把“AI 帮你做一个网页”推进成“你拥有一个能继续推进的项目”：需求能确认，过程能查看，结果能修改，历史能保留，上线由你决定。接下来优先补齐围绕真实文件的 AI Coding 修改闭环、失败恢复和安全编辑，而不是只增加更多 Agent 或外部连接。
