## 背景

Run `0f5ebda1-ec0e-4ae7-9188-33b2eb53dd81` 请求“可以调用本地大模型的翻译软件”。Blueprint 将“调用本地大模型 API”放入已映射需求；Architect 和 Engineer 受全局禁网 Prompt 约束改为模拟翻译；Validator 又因设置说明中出现 `http://` 失败。Repair 删除 URL 后 Validator 通过，但 Reviewer 依据 Blueprint 核心承诺判定模拟翻译和无效 API 配置为 blocker，Run 最终以 `REVIEW_REJECTED` 结束。

## 摘要

- **[P0｜Contract 不可同时满足]** Blueprint 承诺本地 API，而 Engineer 与 Validator 禁止一切网络访问，任何候选都无法同时满足需求和 Runtime 边界。
- **[P1｜Validator 误把所有外网当越界]** 原检查以字符串匹配禁止 `http://`、`https://`、`fetch`、XHR 和 WebSocket，既阻止公网搜索/API，也会把说明文本中的 URL 当成执行能力。
- **[P1｜能力降级不诚实]** 模拟翻译被实现为可操作结果，但 ProductSpec 的核心目标仍是本地模型调用，违反“能力缺口不能伪装完成”的 V1 基线。

## 决策

V1 网络边界调整为：

- 生成应用允许访问公网 HTTP(S)/WS(S) API；推荐 HTTPS，并必须显示 HTTP、CORS 和服务不可用错误。
- 禁止访问 `localhost`、localhost 子域、127/8、`::1`、`0.0.0.0` 和用户设备本地服务。
- “本地大模型”请求进入 `adapted`，本地调用必须列入 `omitted_requirements`，不能继续放在 `mapped_requirements`，也不能用模拟结果冒充完成。
- Agent Provider 本身仍没有网络 Tool；这里开放的是已批准生成应用的浏览器连接能力。

## 2026-07-14 实现 Update

- Product Manager Prompt 已允许公网 API，并明确本地模型/localhost 必须降级和省略。
- Orchestrator 已增加确定性能力归一化：即使 Provider 错误映射本地服务需求，也会在保存 Blueprint 前移入 `omitted_requirements` 并强制标记 `adapted`。
- Architect、Engineer、Engineer Repair Prompt 已从全局禁网改为公网允许、loopback 禁止。
- Validator 已允许外网 URL、fetch、XHR 和 WebSocket，继续拒绝 dynamic import、eval、HTML 注入容器和 loopback 地址。
- Preview CSP 已开放 HTTP(S)/WS(S) `connect-src`；Preview 与物化 `app.js` 均安装 loopback Guard，避免动态配置绕过静态扫描。
- Mock Provider 已把本地模型关键词归入 `adapted`，测试验证公网 fetch 通过、本地地址拒绝和本地模型需求不进入 mapped requirements。

验证证据：`.venv/bin/ruff check another_atom tests`、`.venv/bin/pytest -q` 和 Studio `npm run build` 通过；集成测试验证物化 `app.js` 含 loopback Guard。

尚未完成 Railway 部署后的真实浏览器外网请求、CORS 失败展示和 loopback Guard 验证，因此本 Review 保持在`待办`。
