## 1. 检查结论

普通项目文档纵切已经贯通：文件能力与内容哈希读取 → Markdown 预览或源码查看 → 本地编辑与未保存保护 → 内容 CAS 和 Project 写占用 → 原子替换与精确 Git commit → 幂等结果及启动恢复。Run Artifact 和可发布应用源码没有被文件面板直接改写。

自动化验证覆盖：

- Repository 与 Artifact 列表、读取及用户隔离；
- 文档保存、Git commit、幂等重放和不创建 ProjectVersion；
- 旧哈希冲突不覆盖新内容；
- 应用源码只读；
- Project 写占用与跨用户保存拒绝；
- Git 已提交而操作记录未完成时的启动恢复；
- 全量后端测试、前端 lint 和生产构建。

本 Review 暂不归档，因为完整 Design 还包含源码保存纵切与实际界面验收。

## 2. 已确认边界

- 普通项目文档保存只产生 Git commit，不创建虚假 ProjectVersion，也不改变发布指针。
- `app-spec.json`、`index.html`、`styles.css`、`app.js` 暂时返回 `editable: false`；这是未完成边界，不是已经实现源码编辑。
- Run Artifact 始终只读；客户端能力字段不能绕过服务端路径与文件类型校验。
- Markdown 预览由受控 React 节点生成，不解析 raw HTML；链接只接受 HTTP(S)、`mailto:`、站内绝对路径和锚点。
